6.5 为什么用跳跃表而不是红黑树？

特性 跳跃表 红黑树
实现复杂度 简单 复杂
范围查询 简单 复杂
内存占用 较高（多级指针） 较低
时间复杂度 O(log n) O(log n)

Redis 选择跳跃表的原因：

  1. 实现简单：比红黑树简单很多
  2. 范围查询友好：ZSET 的核心操作（ZRANGE、ZRANK）
  3. 性能足够好：平均 O(log n)
  4. 并发友好：无锁化实现

七、数据结构优化建议

7.1 选择合适的数据类型

需求 推荐类型 原因

优点：

  • 减少键数量
  • 节省内存
  • 支持部分更新

八、总结

8.1 核心原理

类型 底层实现 关键特性
String SDS O(1) 长度、空间预分配
List QuickList 双向链表 + ziplist
Hash HashTable + ZipList 渐进式 rehash
Set HashTable + IntSet 自动去重
Sorted Set SkipList + HashTable O(log n) 查找、范围查询

8.2 编码转换

Redis 会根据数据特征自动选择最优编码：

  • 小数据：紧凑编码（ziplist、intset、embstr）
  • 大数据：标准编码（hashtable、skiplist、raw）

8.3 性能优化

  • 选择合适的数据类型
  • 合理设置编码阈值
  • 避免大 key
  • 监控内存使用
